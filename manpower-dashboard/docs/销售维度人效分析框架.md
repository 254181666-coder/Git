# 销售维度人效分析框架

## 问题背景

基于每日人员销售数据（储值卡办理 + 商品销售），从**任务分配**和**提成机制**两个维度分析人效，目标是：

- 让每个人都有事做
- 让做得好的人挣得多
- 提升整体销售额和利润
- 避免资源垄断和躺平

---

## 一、核心指标体系

### 1.1 个人基础指标

| 指标           | 计算公式           | 含义           |
| ------------ | -------------- | ------------ |
| **总任务完成率**   | 实际完成 / 下达任务    | 衡量个人整体任务完成情况 |
| **储值卡任务完成率** | 储值实际 / 储值任务    | 办卡能力衡量       |
| **单提销售完成率**  | 单提实际 / 单提任务    | 商品推销能力衡量     |
| **储值卡新增**    | 新增卡数 × 平均额度    | 拉新能力         |
| **转卡率**      | 小卡转大卡次数 / 总办卡数 | 客户提升能力       |
| **续卡率**      | 本期续卡数 / 到期卡数   | 客户留存能力       |
| **商品提成占比**   | 商品提成 / 总提成     | 商品推销贡献       |
| **人均单产**     | 总销售额 / 维护客户数   | 客户产出效率       |

### 1.2 团队分布指标

| 指标          | 含义              | 健康标准                   |
| ----------- | --------------- | ---------------------- |
| **任务完成率分布** | 全员任务完成率的频数分布    | 正态分布 ≈ 健康，两极分化 ≈ 不健康   |
| **提成基尼系数**  | 衡量提成分配公平性       | 0.3-0.4 合理，>0.5 贫富差距过大 |
| **机会覆盖度**   | 高价值品类销售机会被多少人共享 | >85% 合理                |
| **客户流转率**   | 本期流转客户数 / 总客户数  | 反映僵硬度，过低说明资源垄断         |

## 二、任务分配分析诊断

### 2.1 常见问题

| 问题          | 现象                    | 后果            |
| ----------- | --------------------- | ------------- |
| **任务拍脑门**   | 任务拍脑袋定，不基于历史数据和门店实际   | 完不成躺平，完成了躺赢   |
| **任务一刀切**   | 所有人任务一样，不考虑能力、岗位、客户资源 | 强者任务不饱和，弱者完不成 |
| **资源垄断**    | 销冠占据大量优质客户，新人没机会      | 新人成长不起来，老员工躺赢 |
| **任务不分解**   | 只有月度任务，没有分解到周/日       | 无法跟踪调整，最后完不成  |
| **任务和提成脱节** | 任务归任务，提成归提成，不挂钩       | 员工没动力         |

### 2.2 诊断方法

```sql
-- 1. 按人统计任务完成率，看分布
SELECT 
  person_name,
  task_completion_rate,
  CASE 
    WHEN completion_rate < 0.6 THEN '低'
    WHEN completion_rate < 0.9 THEN '中' 
    ELSE '高'
  END AS level
FROM person_monthly
ORDER BY completion_rate;

-- 2. 看看是不是老员工垄断了高产出客户
SELECT 
  enter_years,
  COUNT(*) as person_count,
  AVG(task_completion_rate) as avg_completion,
  SUM(total_sales) as total_sales
FROM employee
GROUP BY enter_years
ORDER BY enter_years;

-- 3. 看看任务和实际能力是否匹配
SELECT 
  predicted_task,
  actual_complete,
  (actual_complete - predicted_task) / predicted_task as diff
FROM person_task;
```

### 2.3 优化原则

**① 任务基于历史数据核定，而非拍脑袋**

```
核定任务 = 历史均值 × 调整系数
调整系数 = (1 + 门店增长目标) × 个人能力系数
```

- 能力强、客户多 → 任务相应提高
- 新人、新岗位 → 任务从低开始，逐步提升

**② 任务分级挂钩提成**

```
完成率 < 60%： 基础提成 × 0.8
60% ≤ 完成率 < 90%： 基础提成 × 1.0
完成率 ≥ 90%： 基础提成 × 1.2 + 超额提成
```

**③ 避免资源垄断，强制流转**

- 连续 2 个月完不成任务 × 客户数 > 平均 → 强制流转部分客户给新人
- 高价值客户（大卡）不能都在少数人手里，要求开放流转
- 目标：**让每个人都有机会，让能者多劳也多得**

## 三、提成机制分析诊断

### 3.1 常见问题

| 问题              | 现象                         | 后果             |
| --------------- | -------------------------- | -------------- |
| **平均主义**        | 大家提成比例一样，不管卖什么             | 没人推高利润商品       |
| **单提提成和办卡提成脱节** | 办卡提成远高于商品                  | 没人推商品，超市没产出    |
| **高利润品没人推**     | 提成比例一样，推起来费劲               | 大家都推容易卖的，利润上不来 |
| **老员工躺赢**       | 老客户躺着也有提成，不用开发新客户          | 没人开发新客户，增长停滞   |
| **贴头皮办卡**       | 为了提成，抢着给快消费完的客户办卡，同一个客户反复办 | 客户反感，影响体验      |

### 3.2 优化设计方案

#### 方案一：分级提成制（推荐）

**核心逻辑：** 不同品类不同提成比例，引导销售推我们想让推的

| 品类               | 提成比例  | 目的           |
| ---------------- | ----- | ------------ |
| 储值卡（新办）          | **高** | 鼓励办卡         |
| 储值卡（转卡/续卡）       | 更高    | 鼓励提升客户       |
| 高利润商品（果盘/礼炮/第三方） | 中高    | 鼓励推销高利润      |
| 低利润流通商品（纸巾/矿泉水）  | 低     | 不鼓励重点推，但卖了也有 |
| 新品               | 临时提高  | 鼓励推广试错       |

#### 方案二：任务+提成二级挂钩

```
总提成 =
  (储值任务完成 × 储值提成率) +
  (单提任务完成 × 单提提成率) +
  超额部分 × 超额提成率
```

**举例**：

- 经理任务：储值 10万 + 单提 5万
- 完成储值提 2%，完成单提提 6%
- 超额完成：储值提 3%，单提提 8%
- 激励大家多卖，多卖多拿

#### 方案三：团队协作提成

**核心问题解决：** 避免"有人累死、有人闲死"

- 迎宾：拉新进店 → 办成卡拿 **引荐提成**（1%）
- 经理：跟进服务最终办卡 → 拿 **主要提成**（2%）
- 导购：超市商品推销 → 拿 **商品提成**
- 每个人做自己擅长的，每个人都有提成，避免"一个人全干"

#### 方案四：新客额外激励

```
新客户开发 = 基础提成 + 额外奖励
- 新客首办，额外加 50 元/张
- 连续三个月新客开发达标，底薪升级
```

鼓励员工不停开发新客户，避免吃老本。

### 3.3 公平性保障：机会均等算法

**核心问题：** 避免优质销售机会都被少数人垄断

```python
def opportunity_fairness(position, category):
    # 获取该员工最近7天该品类销售次数
    sales_count = get_recent_sales_count(position, category)
    max_count = get_max_sales_count(category)

    # 机会权重：卖得越少，权重越高（鼓励轮转）
    weight = 1 - (sales_count / max_count)

    # 高价值品类需要机会均衡
    if category in ['储值卡', '高价值礼炮'] and weight > 0.6:
        return True  # 分配机会
    else:
        return False # 暂停分配
```

**规则：**

- 高价值机会轮流来，不能一个人全占了
- 卖得少的员工优先获得新机会
- 保证每个人都有成长机会

## 四、分析流程（每日数据可落地）

### 步骤1：数据整理

你每天提供的数据需要包含：

```
日期 | 人员姓名 | 岗位 | 储值卡办理数 | 储值卡金额 | 商品销售额 | 商品提成 | 续卡数 | 转卡数
```

### 步骤2：周/月汇总计算指标

```
for 每个人:
  计算：
    任务完成率
    储值贡献占比
    商品贡献占比
    新增客户数
    续卡率/转卡率

for 整个门店:
  计算：
    任务完成率分布
    提成基尼系数
    机会覆盖度
```

### 步骤3：识别问题

| 识别           | 问题     | 调整方向          |
| ------------ | ------ | ------------- |
| 多数人完成率 < 60% | 任务定高了  | 降任务           |
| 基尼系数 > 0.5   | 贫富差距太大 | 调整任务分配，强制流转客户 |
| 商品提成占比 < 20% | 商品推销不够 | 提高商品提成比例      |
| 新客开发数/人 < 目标 | 没人开发新客 | 新客额外加激励       |

### 步骤4：持续调整

- 每周一小结，调整不合理的任务
- 每月一总结，调整提成比例
- 每季度重新分配客户，避免垄断

## 五、总结：健康的人效长什么样

✅ **好的状态**

- 任务完成率平均在 70%-80%，大部分人能完成，少数人需要努力
- 基尼系数在 0.3-0.4 之间，差距有但不过大
- 机会覆盖度 > 85%，新人有机会
- 商品提成占比在 20%-40%，超市能贡献利润
- 持续有新客户开发，客户池子不断扩大

❌ **需要调整的信号**

- 任务完成率两极分化，要么全完成要么全不完成
- 基尼系数 > 0.5，少数人拿大部分提成
- 老员工霸占优质客户，新人不开胡
- 商品提成占比太低，超市没人推
- 新客户开发长期不达标，池子越来越小

## 六、落地建议

基于你现在每天都能提供数据表，建议这么落地：

1. **先算指标**：用上面的框架先算一遍当前状态，看看问题出在哪
2. **局部调整**：先调一两个问题（比如任务核定、商品提成），看效果
3. **逐步推开**：见效了再推广到任务分配、客户流转这些更深的调整
   4 **持续跟踪**：每周看数据，不好就调，找到适合你门店的平衡点

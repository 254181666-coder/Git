# 人效管理模型完整设计

## 一、核心目标

通过数据化管理，实现：

1. **人尽其才** - 每个员工做擅长的事，有合适的任务量
2. **多劳多得** - 干得多挣得多，激励员工
3. **机会公平** - 新人有机会成长，不会被老员工垄断资源
4. **持续优化** - 每月诊断，持续调整，不会一潭死水

## 三、核心计算指标

### 3.1 员工个人指标

| 指标        | 计算公式             | 含义         | 健康范围       |
| --------- | ---------------- | ---------- | ---------- |
| **任务完成率** | `实际完成 ÷ 下达任务`    | 衡量个人任务完成情况 | 平均 70%-90% |
| **储值完成率** | `储值实际 ÷ 储值任务`    | 办卡任务完成     |            |
| **商品完成率** | `商品实际 ÷ 商品任务`    | 商品推销任务完成   |            |
| **人均产出**  | `总销售额 ÷ 维护客户数`   | 每个客户平均产出   | 越高越好       |
| **新客开发数** | 月度新增有效客户数        | 衡量拉新能力     |            |
| **续卡率**   | `续卡数 ÷ 到期卡数`     | 客户留存能力     | 越高越好       |
| **转卡率**   | `小卡转大卡次数 ÷ 总办卡数` | 客户提升能力     | 越高越好       |

### 3.2 门店整体指标

| 指标          | 计算公式             | 含义         | 健康范围            |
| ----------- | ---------------- | ---------- | --------------- |
| **提成基尼系数**  | 计算所有员工提成的基尼系数    | 衡量提成分配公平性  | **0.3 - 0.4**   |
| **机会覆盖率**   | `有销售员工数 ÷ 总员工数`  | 衡量机会分配公平性  | **≥ 85%**       |
| **客户流转率**   | `月度流转客户数 ÷ 总客户数` | 衡量客户流转是否活跃 | 月度 **5% - 10%** |
| **零销售员工占比** | `零销售员工数 ÷ 总员工数`  | 衡量人浮于事     | **< 10%**       |
| **平均任务完成率** | 所有员工任务完成率平均      | 衡量任务设定是否合理 | 70% - 90%       |
| **商品提成占比**  | `商品提成总额 ÷ 总提成`   | 衡量超市贡献     | **20% - 40%**   |

## 四、人效调整规则

### 4.1 任务核定规则

**基础公式：**

```
个人月度任务 = 历史均值 × (1 + 门店增长目标) × 能力系数
```

- **历史均值**：过去 3 个月个人平均完成
- **增长目标**：门店整体增长目标，比如 5%
- **能力系数**：
  - 新人（< 3 个月）：`0.5 - 0.8` 慢慢往上加
  - 成熟员工：`1.0`
  - 优秀员工：`1.1 - 1.2` 挑战更高目标

**分解：**

```
年度目标 → 分解到月度 → 分解到个人
月度任务 → 分解到周 → 分解到天
每天追踪，每周调整
```

### 4.2 提成核定规则

**核心原则：**

```
高利润商品 > 低利润商品 → 提成比例高
推新品 → 临时提高提成
卖得多 → 提成比例递增
```

**分级提成比例示例：**

| 品类               | 提成比例     | 目的      |
| ---------------- | -------- | ------- |
| 储值卡（新办/转卡）       | 2% - 3%  | 激励办卡    |
| 高利润商品（果盘/礼炮/第三方） | 6% - 8%  | 激励推销高利润 |
| 日常商品（酒水/饮料）      | 2% - 4%  |         |
| 流通小商品（纸抽/矿泉水）    | 1% - 2%  |         |
| 新品推广             | 临时 + 50% | 鼓励试错推广  |

### 4.3 机会公平规则

**机会均等算法（解决资源垄断）：**

```python
def can_assign_opportunity(employee, category):
    # 该员工最近 7 天该品类销售次数
    sales_count = get_recent_sales(employee, category)
    max_count = get_max_count(category)

    # 卖得越少，权重越高，越优先获得机会
    weight = 1 - (sales_count / max_count)

    if weight > 0.6:  # 阈值
        return True  # 分配机会
    else:
        return False # 暂停分配，给其他人机会
```

**强制流转规则（解决老员工垄断）：**

- 连续 2 个月任务完成率 < 60%
- 手里客户数 > 门店均值 × 1.5
- ⇒ 强制流转出部分客户给新人

### 4.4 考核调整节奏

```
每日：更新销售数据，员工可看自己进度
每周：店长复盘调整，解决本周问题
每月：全店整体人效诊断，调整任务/提成/人员
季度：结构性调整（编制/岗位）
```

## 五、诊断决策树

每次月度诊断，按这个流程决策：

```
开始
├─ 机会覆盖率 < 85% ?
│    ⇒ 问题：机会集中在少数人
│    ⇒ 解决：强制流转，分配部分客户给新人
│
├─ 基尼系数 < 0.3 ?
│    ⇒ 问题：过于平均，能者不能多得
│    ⇒ 解决：拉大提成差距，多卖多提
│
├─ 基尼 > 0.4 ?
│    ⇒ 问题：贫富差距过大，多数人没机会
│    ⇒ 解决：调整任务分配，强制机会分流
│
├─ 商品提成占比 < 20% ?
│    ⇒ 问题：超市推销不足，利润流失
│    ⇒ 解决：提高商品提成比例，激励员工推销
│
└─ 零销售员工占比 > 10% ?
     ⇒ 排查原因：
     ├─ 后勤岗位 → 正常
     ├─ 一线岗位 → 谈话分析原因
     │     └─ 不能改进 → 调整岗位 / 淘汰
```

## 六、落地步骤

### 第一步（第 1-2 周）

- 整理历史数据，补齐缺失数据
- 核定每个人月度任务：储值+商品

### 第二步（第 1 个月）

- 按新模型执行
- 每日更新数据，店长每周复盘

### 第三步（第一个月结束）

- 输出诊断报告
- 调整不合理的任务/提成
- 处理零产出员工

### 第四步（持续）

- 每月重复诊断调整
- 持续优化

## 八、核心一句话总结

> **数据化 → 分解到个人 → 每日追踪 → 月度诊断 → 持续调整**

> 让**"能者多劳 → 多劳多得 → 多得就能留住能人 → 能人带来更高增长"** 形成良性循环

## 三、关键业务逻辑详解

### 3.1 单位换算逻辑
**重要原则**：数据库中金额单位为「分」，展示时转换为「元」

```python
# 数据库值（分） → 展示值（元）
display_value = db_value / 10

# 关键：卡级判断用「原始分值」，展示时才转元
# ✅ 正确做法：先判断卡级，再转元
level = get_level_from_amount(amount_in_fen)  # 用分判断
display_amount = amount_in_fen / 10           # 最后才转元

# ❌ 错误做法：先转元导致卡级判断错误
level = get_level_from_amount(amount_in_fen / 10)  # 会把1314变成131.4
```

### 3.2 客单价逻辑
| 年份 | 数据来源字段 | 说明 |
|------|------------|------|
| 2025年 | `efficiency` | 直接使用2025年收入信息表中的客单价 |
| 2026年 | `daily_batch_consumption` | 直接使用日营业数据表中的日单批消费 |

### 3.3 储值卡卡级判断逻辑
| 年份 | 判断字段 | 说明 |
|------|---------|------|
| 2025年 | `stored_amount` | 储值金额 |
| 2026年 | `drink_principal` | 酒水本金 |

**卡级区间（2026-04-29 确认）**：
```
1-300, 300-500, 500-1000, 1000-2000, 2000-3000, 3000-5000, 5000-10000, 10000+
```

### 3.4 日期处理逻辑
```python
# 横轴只显示日（不用月-日）
df['日期'] = df['data_date'].dt.strftime('%d')

# 排序时要转数字，避免 1, 10, 11... 的字符串排序错误
df['日期_num'] = df['日期'].astype(int)
df = df.sort_values('日期_num')

# Plotly设置X轴为分类轴，确保正确显示
fig.update_xaxes(type='category')
```

### 3.5 图表配色逻辑
- **2025年**: 蓝色 `#1f77b4`
- **2026年**: 橙色 `#ff7f0e`
- 原则：撞色对比，清晰区分

## 四、同比分析页面核心逻辑

### 4.1 数据加载（src/components/year_over_year.py）
```python
@st.cache_data(ttl=3600)
def load_store_daily_data():
    """加载门店日报数据"""
    sql = """SELECT store_id, data_date, 
                    COALESCE(total_revenue, revenue) as revenue,
                    total_revenue, customers, efficiency, daily_batch_consumption,
                    stored_card_sales, times_card_sales
             FROM store_daily WHERE data_date >= '2025-01-01'"""
    return query(sql)

@st.cache_data(ttl=3600)
def load_stored_value_data():
    """加载储值数据"""
    sql = """SELECT store_id, data_date, stored_amount, drink_principal,
                    member_level, stored_count
             FROM stored_value WHERE data_date >= '2025-01-01'"""
    return query(sql)
```

### 4.2 数据筛选流程
```
1. 用户选择：门店 + 月份
2. 按年份拆分：2025年数据 vs 2026年数据
3. 营业额转元：df['revenue'] / 10
4. 客单价映射：
   - 2025年 → efficiency
   - 2026年 → daily_batch_consumption
5. 储值数据：保留原始值，最后才转元
```

### 4.3 储值卡统计逻辑
```python
# 1. 定义卡级区间
card_intervals = [
    (0, 300, '1-300'),

# 2. 2025年用 stored_amount 判断卡级

# 4. 最后展示时才转元
```

## 六、常见问题排查

### 6.1 2025年数据显示为0
**原因**：先除以10转元，再判断卡级  
**修复**：先用原始分值判断卡级，最后展示时才转元

### 6.2 横轴日期排序错误
**原因**：字符串排序导致 "1, 10, 11, 2..."  
**修复**：
```python
df['日期_num'] = df['日期'].astype(int)
df = df.sort_values('日期_num')
fig.update_xaxes(type='category')
```

### 6.3 客单价数据对比离谱
**原因**：2025年/2026年用错了字段  
**修复**：
- 2025年 → `efficiency`
- 2026年 → `daily_batch_consumption`

### 6.4 数据库连接失败
**检查**：
1. MySQL服务是否启动
2. src/config.py 中 `USE_MYSQL` 是否为 True
3. `MYSQL_CONFIG` 配置是否正确

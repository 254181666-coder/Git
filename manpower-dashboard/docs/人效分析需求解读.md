# 人效分析项目 - 需求解读

## 一、项目背景

这是糖果华庭KTV连锁企业（12家店面）的人效分析优化项目，基于完整的会议记录整理。

企业当前面临市场下行压力，需要通过人效优化来：

- 控制人力成本
- 提升人均产出
- 实现精细化管理
- 稳定利润水平

## 二、核心问题诊断

从会议记录中提炼出的核心人效问题：

| 问题类别        | 具体表现                     |
| ----------- | ------------------------ |
| **编制不合理**   | 人浮于事，岗位饱和度过低，存在冗余编制      |
| **任务分配失衡**  | 老员工霸占优质客户资源，新客户/新员工得不到机会 |
| **提成机制不合理** | 平均主义，干多干少提成差距不大，激励不足     |
| **岗位分工不清晰** | 多个员工争抢同一个客户，或者无人负责       |
| **资源分配不公平** | 优质客户资源集中在少数人手里，大部分员工没机会  |
| **能力评价缺失**  | 无法量化评估每个员工/岗位的实际贡献       |
| **人效计算粗糙**  | 只看总额，不看人均产出、单位时间产出       |

## 三、人效分析核心指标体系

### 1. 基础指标

- **总人力成本** = 底薪 + 提成 + 保险 + 福利 + 其他补助
- **总产出** = 办卡总额 + 商品销售总额 + 团购总额
- **人均产值** = 总产出 / 总人数
- **人均利润** = (总产出 - 总人力成本 - 分摊成本) / 总人数

### 2. 人效效率指标

- **岗位饱和度** = 实际完成任务 / 额定任务
  - > 1.2 过度饱和，考虑增编
  - < 0.8 不饱和，考虑减编
- **任务完成率** = 实际完成任务 / 下达任务
- **客户承载量** = 维护客户数 / 员工额定承载
- **工时利用率** = 有效工作时长 / 总工时

### 3. 销售转化指标

- **新客开发率** = 新开发客户数 / 总客户数
- **续卡率** = 本期续卡数 / 到期卡数
- **转卡率** = 小卡转大卡数 / 总办卡数
- **提卡成功率** = 成功提卡数 / 邀约客户数

### 4. 公平性指标

- **机会覆盖度** = Σ(各岗位销售主攻品类次数) / Σ(该品类总销售次数)
  - 目标值 ≥ 85%，低于则需轮岗训练
- **贡献基尼系数** - 衡量任务/提成分配的均衡性

## 四、人效优化模型

### 1. 动态调编模型

```
岗位饱和系数 = (实际销售额 ÷ TOP10员工均值) × (客户接触时长 ÷ 班次时长)

规则：
- 系数 > 1.2 → 自动缩减1人编制
- 系数 < 0.8 且持续5个工作日 → 增加1人轮岗
- 每日闭店后系统自动输出编制调整建议
```

### 2. 机会均等算法

```python
    weight = 1 - (position.sales_count / max_sales_count)

```

### 3. 任务测算模型

```
新增储值卡任务 = 基础任务 × 激活系数 × 新客系数

激活系数 = 1 + (沉睡卡数 × 0.2 ÷ 总卡数)
新客系数 = 1 + (3月新客流失率 × 0.5)

示例：
某店有1000张沉睡卡（总卡5000），3月新客流失率20%
任务量 = 50 × (1+0.04) × (1+0.1) = 57张
```

### 4. 岗位-商品绑定矩阵（通辽标准）

| 岗位    | 主攻品类（提成70%）   | 辅攻品类（提成30%）  | 禁售品类   |
| ----- | ------------- | ------------ | ------ |
| 导购    | 礼炮、点单商品（鸭货等）  | 储值卡          | 酒水饮品   |
| 经理/助理 | 储值卡、氛围组（包厢套餐） | 高单价礼炮（>200元） | 低价果盘   |
| 迎宾    | 储值卡（新开卡）      | 礼炮引流款        | 需加工的商品 |
| 服务生   | 二次促销商品（小吃、果盘） | 即饮类商品        | 储值卡    |
| 超市吧员  | 预包装商品（明太鱼等）   | 果盘           | 礼炮     |

## 五、客户标签体系

### 1. 符号化编码规则（适配微信备注）

- **格式**：`[来源2位][性别1位][星级1位][关系码2位] [动态标签]`
- **总长度**：基础8位 + 动态≤20字符，符合微信备注限制

**来源编码：**

- TG = 团购
- MB = 陌拜
- CK = 储值卡老客
- ZB = 转介绍

**性别编码：**

- M = 男
- F = 女

**星级编码（卡级）：**

- S = S级 (≥5000)
- A = A级 (2000-5000)
- B = B级 (500-2000)
- C = C级 (<500)
- R = 团购客

**关系码：**

- 01 = 无介绍
- 02 = 李刚介绍
- ... 以此类推

**动态标签优先级：**
生日/余额 → 性格 → 消费频次 → 商品偏好

**示例：**

- `MBMSR2 B0910Y3 JHJ` = 陌拜男客户 S级 马总介绍 阳历生日0910 余额>2k 性格好面子 偏好青岛啤酒
- `TGFCR0 B1225 F0` = 团购女客 C级 无介绍 阳历生日1225 低频消费

### 2. 维护分级策略

| 级别           | 维护策略                                          |
| ------------ | --------------------------------------------- |
| **S级高价值VIP** | 现场专属服务（高管敬酒、定制布置）+ 年节专属礼遇（端午/中秋/生日礼 + 2次随机惊喜） |
| **A级稳定客户**   | 定向微信维护（余额提醒、节日问候）+ 赠送小额福利（果盘、存酒提醒）            |
| **B级潜力新客**   | 系统后台推送优惠活动 + 生日特惠激活                           |
| **C级团购客**    | 系统后台统一推送限时优惠 + 拉进私域群                          |

升级规则：客户升级后立即启动上级服务待遇。

## 六、积分体系设计

### 1. 积分获取

- 每消费 **10元** = **1积分**（储值卡本金+赠金都算）
- 团购按实际消费额积分
- 次卡按三次消费计算积分（二赠一活动）
- 特定商品（香烟、槟榔）不积分

**活动加成：**

- 店庆/特定节日：双倍/十倍积分
- 淡旺季不同倍率

### 2. 积分有效期

- 按年清零，每年12月31日清零

### 3. 积分兑换（示例）

| 积分区间      | 可兑换礼品          |
| --------- | -------------- |
| 100-500   | 纸抽、爆米花         |
| 500-1000  | 果盘、啤酒          |
| 1000-2000 | 参泉、口红、吹风机      |
| 2000-5000 | 车载香氛、视频会员、娃娃   |
| 5000+     | 海参、鹿茸、人参、手机、茅台 |

### 4. 积分应用

- 积分换礼品用于 **提卡升级** 钩子
- 历史积分 + 新储值 = 兑换礼品 + 升级卡级

## 七、数据分析输出

### 1. 店面人效诊断报表（月度）

- 各岗位人均产出排名
- 饱和度分析，标注哪些岗位该增编/减编
- 提成分配公平性分析
- 新客开发/续卡/转卡转化率
- 同比/环比变化趋势

### 2. 员工能力雷达图

五个维度打分可视化：

1. 任务完成度（卡）
2. 商品单提（销售能力）
3. 老客户维护能力
4. 新客户拉新能力
5. 升卡提卡成功率

### 3. 人效优化建议

- 哪些岗位/人员编制冗余，建议减编
- 哪些岗位负荷过大，建议增编
- 提成机制调整建议
- 任务分配优化建议
- 客户资源重新分配建议

## 八、项目实施路径

1. **第一步**：数据采集 - 收集各店面现有编制、薪资、任务、销售数据
2. **第二步**：现状诊断 - 计算各项人效指标，找出问题
3. **第三步**：方案设计 - 编制调整、提成机制优化、岗位重新分工
4. **第四步**：试点验证 - 选择1-2个店面试点运行3个月
5. **第五步**：全面推广 - 总结经验后推广到全部门店
6. **第六步**：持续监控 - 月度跟踪人效指标变化，持续优化

## 九、核心价值预期

通过人效分析和优化，预期可以实现：

- **人力成本降低**：10%-15%（通过减编冗余岗位）
- **人均产值提升**：15%-20%（通过合理任务分配和激励）
- **利润提升**：预计提升8%-12%的净利润
- **管理精细化**：从"拍脑门"到"数据驱动"决策

# 第一阶段数据口径

本文件用于核对 API 返回字段是否覆盖抖音来客后台常用导出表。

## 已核销明细

必须确认的字段：

- 商家主体
- 门店名称
- 门店 ID 或 POI ID
- 订单 ID
- 券码或券 ID
- 核销时间
- 核销状态
- 商品/套餐名称
- SKU 或套餐 ID
- 用户实付金额
- 平台补贴金额，若有
- 商家应收或结算金额，若接口提供
- 核销操作来源，若接口提供

## 退单明细

必须确认的字段：

- 商家主体
- 门店名称
- 订单 ID
- 券码或券 ID
- 退款单 ID
- 退款申请时间
- 退款成功时间
- 退款状态
- 退款金额
- 原订单金额
- 商品/套餐名称
- 退款原因，若接口提供

## 套餐/商品内容

必须确认的字段：

- 商家主体
- 商品 ID
- SKU ID
- 套餐名称
- 套餐内容
- 售价
- 库存，若接口提供
- 适用门店
- 上下架状态
- 生效时间
- 失效时间

## 门店基础信息

必须确认的字段：

- 商家主体
- 门店名称
- 门店 ID 或 POI ID
- 地址
- 城市/区域
- 营业状态，若接口提供
- 与来客后台门店名称是否一致

## 第二阶段：员工短视频数据

第一阶段只确认接入条件。后续需要的数据口径：

- 员工账号
- 绑定关系：品牌号/员工号/合作号
- 视频发布时间
- 视频标题或视频 ID
- 播放量
- 点赞数
- 评论数
- 分享数
- 主页访问数
- 统计日期

注意：部分主页数据接口可能有授权、延迟和粉丝量限制，不能把当天实时数据作为第一版验收标准。

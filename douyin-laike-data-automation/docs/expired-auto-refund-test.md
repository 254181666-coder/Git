# 发券成功-过期自动退测试记录

## 场景目标

验证商家券发券成功后，用户购买 2 份、商家只核销其中 1 份、商品有效期到期触发过期自动退时，API 是否能完整追踪订单状态、券状态和退款/退单状态。

## 前置条件

- 商家系统已创建一个团购券商品。
- 券码类型选择 `商家券`。
- 商品有效期设置为 1 天。
- 券码服务商和商家系统商品 ID 已正确关联。
- 抖音侧可正常购买该商品并完成支付。
- 当前商家主体已完成开放平台授权，具备订单、券状态、核销、退款相关接口权限。

## 执行步骤

1. 在商家系统创建商品并同步到抖音。
2. 在抖音端购买 2 份并完成支付。
3. 记录订单 ID、商品 ID、SKU ID、2 个券码或券 ID、支付时间和支付金额。
4. 商家系统发券，确认发券成功。
5. 商家只核销其中 1 张券，保留另 1 张未核销。
6. 等商品有效期到期后，等待抖音发起过期自动退。
7. 调用订单查询接口，确认订单状态是否体现部分核销、部分退款或最终状态。
8. 调用券状态查询/批量查询接口，确认 2 张券分别为已核销、已退款/退款完成。
9. 调用退款/退单查询接口，确认未核销券生成退款记录。
10. 若有回调/SPI，检查回调日志是否收到核销、退款或状态变更通知。

## 验收标准

- 订单查询能通过订单 ID 找到该笔订单，并返回订单状态、金额和商品信息。
- 券状态查询能区分同一订单下不同券的状态。
- 已核销券状态为已核销，并包含核销时间或可关联到验券历史。
- 未核销券在过期自动退后变为退款中、已退款或等价状态。
- 退款/退单查询能返回退款单 ID、退款金额、退款时间、原订单 ID 和券 ID/券码关联关系。
- 商家系统、抖音来客后台、API 返回结果三方状态一致，或差异有明确延迟说明。

## 注意事项

- 商品有效期为 1 天时，自动退不一定在过期瞬间完成，现场需要记录实际触发时间。
- 如果订单查询只返回订单级状态，必须用券状态或退款接口补齐单券维度。
- 如果 API 返回空数据，先确认查询时间范围、商家授权主体、订单归属门店和接口权限。

# 授权试点执行手册

## 试点范围

- 日期：2026-05-17
- 范围：先选 1 家独立商家主体
- 目标：完成授权，并验证门店、订单、核销、退款、商品/套餐数据可访问

## 现场准备

试点店负责人需要准备：

- 抖音来客商家主体名称
- 来客账号负责人姓名
- 可接收短信或扫码确认的手机号
- 店铺/门店名称
- 营业执照主体信息，若授权流程要求核验
- 可配合现场扫码、短信、登录确认的人员

技术侧需要准备：

- 抖音开放平台账号
- 生活服务商家应用
- 应用的 Client Key、Client Secret、回调地址
- 已申请或待申请的生活服务能力清单
- 接口调试工具或后续程序环境

## 推荐执行顺序

1. 在 `data/authorization-ledger.csv` 中把 12 家店的基础信息先填完整。
2. 标记其中 1 家为 `试点中`。
3. 登录抖音开放平台，确认生活服务商家应用已创建。
4. 申请或确认以下能力：
   - 门店信息查询
   - 订单查询
   - 验券历史查询
   - 券状态查询
   - 退款/退单相关查询
   - 商品/套餐线上数据查询
   - 分账明细查询，若需要财务对账
5. 由试点商家负责人完成商家授权。
6. 在开放平台后台确认该商家主体显示为已授权。
7. 用试点主体调用 `data/api-verification-checklist.csv` 中的接口项。
8. 记录每个接口的响应状态、数据覆盖情况和问题。
9. 若所有核心数据可访问，把试点店状态改为 `已授权`。
10. 把流程复制给其余 11 家，逐家授权。

## 验收标准

第一天完成即算成功：

- 1 家试点商家主体授权成功。
- 门店信息接口可正常返回。
- 订单或核销接口至少有一个能返回正常响应，真实数据或空数据都可以。
- 商品/套餐数据能确认接口路径和字段来源。
- 授权流程中的阻塞点已记录，方便复制到另外 11 家。

## 员工短视频数据

第一阶段不接入员工短视频数据，只确认：

- 试点店是否有品牌号。
- 员工账号是否能绑定为品牌员工号。
- 员工账号负责人是否能配合授权。
- 是否存在千粉以下账号，避免后续接口取不到部分主页数据。

## 备用方案

如果某类数据官方 API 暂时拿不到，再考虑后台导出：

- 已核销明细 Excel
- 退单明细 Excel
- 套餐/商品内容 Excel

后台自动化只做备用，因为登录、验证码、扫码和页面改版都会影响稳定性。

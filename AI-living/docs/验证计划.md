# 播助手 Pro - 技术验证计划

## 验证目标

在投入大量时间和资金前，验证核心技术是否可行，降低项目风险。

---

## 阶段一：基础技术验证（不需要GPU）

### 1.1 OpenAI API集成验证

**目标**: 验证AI智能回复功能是否可用

**测试步骤**:
1. 配置OpenAI API Key
2. 测试基础聊天功能
3. 测试商品信息配置
4. 测试直播场景回复质量

**预期结果**:
- ✅ API调用正常
- ✅ 回复质量满足直播场景
- ✅ 响应时间 < 2秒

**成本**: 约¥0.1-0.5（测试用量）

**时间**: 1小时

---

### 1.2 FFmpeg RTMP推流验证

**目标**: 验证能否成功推流到直播平台

**测试步骤**:
1. 安装FFmpeg
2. 准备测试视频文件
3. 推流到测试RTMP服务器
4. 验证画面质量、延迟

**预期结果**:
- ✅ 推流成功，无报错
- ✅ 画面清晰，延迟 < 3秒
- ✅ 支持1080x1920竖屏

**成本**: ¥0（使用本地测试）

**时间**: 2小时

**验证命令**:
```bash
# 测试推流到本地RTMP服务器
ffmpeg -re -i test.mp4 -c:v libx264 -c:a aac -f flv rtmp://localhost/live/test
```

---

### 1.3 话术轮播验证

**目标**: 验证话术调度逻辑是否正常

**测试步骤**:
1. 添加测试话术
2. 启动轮播
3. 观察切换逻辑
4. 验证防检测效果

**预期结果**:
- ✅ 话术按设定顺序播放
- ✅ 切换平滑，无卡顿
- ✅ 防检测随机暂停生效

**时间**: 1小时

## 阶段二：SadTalker唇形驱动验证（需要GPU）

### 2.1 阿里云GPU实例准备

**目标**: 部署SadTalker服务

**推荐配置**:
- 实例规格: `ecs.gn6v-c8g1.2xlarge` (T4 16G)
- 系统: Ubuntu 22.04 LTS
- 付费模式: 抢占式实例（约¥0.5/小时）
- 存储: 100G系统盘

**预计成本**: ¥0.5/小时（测试2小时约¥1）

**部署时间**: 2-3小时（首次）

---

### 2.2 SadTalker功能验证

**目标**: 验证唇形驱动效果

**测试步骤**:
1. 上传测试人脸照片
2. 输入测试文本
3. 生成唇形同步视频
4. 评估效果质量

**预期结果**:
- ✅ 唇形同步准确
- ✅ 生成速度可接受（< 30fps）
- ✅ 画面自然，无明显违和感

**验证指标**:
- 唇形准确度
- 生成速度（帧/秒）
- 画面质量

---

### 2.3 性能测试

**目标**: 测试单GPU能支持几路并发

**测试步骤**:
1. 单路推流测试
2. 双路并发测试
3. 三路并发测试
4. 观察GPU占用、生成速度

**预期结果**:
- ✅ 单路: GPU占用 < 50%
- ✅ 双路: GPU占用 < 80%
- ✅ 三路: 可能卡顿，评估是否可行

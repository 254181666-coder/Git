# 每日报告 - 自动化生成说明

## 当前流程

`run_daily_report.sh` 现在是统一入口：

1. 同步 Fun360 数据到本地 SQLite
2. 检查 `store_daily` 和 `product_sales_summary` 是否已有目标日期数据
3. 生成轻舟日报图表、商品销售报告、同比对比报告
4. 复制标准交付文件到 `DAILY_REPORT_DELIVERY_DIR` 或 `data/delivery`

默认生成昨天的营业日数据，营业日口径为 08:00 到次日 08:00。

## 手动运行

```bash
cd /Users/huatingyule/Desktop/每日报告
./run_daily_report.sh
```

指定日期：

```bash
./run_daily_report.sh 2026-05-17
```

仅使用本地数据库、不拉接口：

```bash
./run_daily_report.sh 2026-05-17 --no-sync
```

数据不完整时强制继续：

```bash
./run_daily_report.sh 2026-05-17 --allow-missing-data
```

## 环境变量

Fun360 同步需要：

- `FUN360_APPID`
- `FUN360_SECRET`
- `FUN360_PHASE`，可选，默认 `production`
- `FUN360_TOKEN`，可选
- `FUN360_AUTHORIZATION`，可选

交付目录可选配置：

```bash
export DAILY_REPORT_DELIVERY_DIR="/Users/huatingyule/每日日报"
```

如果需要临时跳过接口同步：

```bash
export DAILY_REPORT_SYNC=0
```

## 定时运行

项目内仍保留调度器：

```bash
python3 run_tasks.py stop
python3 run_tasks.py start
```

默认每天 10:10 运行日报，23:00 清理归档文件。

## 涉及文件

| 文件                    | 说明                |
| --------------------- | ----------------- |
| `scripts/sync_fun360_daily_data.py` | 同步 Fun360 门店、商品销售、会员储值 |
| `scripts/daily_report.py` | 同步、数据检查、报告生成、复制交付 |
| `run_daily_report.sh` | 统一运行入口，优先使用带依赖的 Python |
| `run_tasks.py` | 定时调度入口 |

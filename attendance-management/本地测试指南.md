# 本地测试指南

本文档说明如何进行本地测试，包含两人的实际测试数据。

## 快速开始

## 运行入口冻结说明

当前测试版本统一使用根目录 `main.py` 作为运行入口。

请使用：

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

暂不要使用：

```bash
uvicorn app.main:app
```

`app/` 目录是模块化迁移草稿，迁移未完成，明天本地数据测试请以根目录 `main.py` / `models.py` 的逻辑为准。

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 初始化测试数据

```bash
python init_test_data.py
```

这将创建：
- 2个店面（朝阳门店、海淀门店）
- 2个测试员工
- 近30天的考勤记录
- 上个月的工资记录

### 3. 启动服务

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

### 4. 访问应用

- 应用首页: http://localhost:8000
- API 文档: http://localhost:8000/docs
- 交互式 API: http://localhost:8000/redoc

## 测试场景建议

### 场景1: 员工管理
1. 使用 admin 账号登录
2. 查看现有员工列表
3. 尝试编辑员工信息
4. 添加新员工测试

### 场景2: 考勤记录
1. 查看张三和李四的考勤记录
2. 注意李四有一次迟到记录
3. 张三有一次早退记录
4. 尝试添加新的考勤记录

### 场景3: 工资管理
1. 查看上个月的工资记录
2. 检查工资计算是否正确
3. 尝试生成新的工资记录

### 场景4: 请假管理
1. 查看李四的请假记录
2. 尝试提交新的请假申请
3. 审批请假申请

## 重置数据

如需重新生成测试数据：

```bash
python init_test_data.py reset
```

## 数据库

测试使用 SQLite 数据库，数据文件保存在项目根目录下的 `attendance.db`。

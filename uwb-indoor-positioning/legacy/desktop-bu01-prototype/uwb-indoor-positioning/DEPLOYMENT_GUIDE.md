## 1. 项目概述

### 1.1 项目目标
- **场景**: 4000平门店室内人员/物品定位
- **功能**: 实时定位 + 轨迹追踪 + 历史回放
- **精度**: 10~30cm

### 1.2 系统架构

```
UWB标签 → UWB基站 → CP2102 → USB集线器 → Python后端 → MySQL → Web前端
```

### 1.3 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 硬件 | NodeMCU-BU01 + CP2102 | UWB定位模块 + 串口转换 |
| 后端 | Flask + Python | 定位算法 + API服务 |
| 数据库 | MySQL | 位置数据存储 |
| 前端 | ECharts | 轨迹可视化 |

```
│  └──────────┘    └──────────┘    │    │                     │
│                                  │    │                     │
│                          ┌───────┴────┴───────┐             │

│                          └─────────────────────┘             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 引脚连接表

| CP2102引脚 | BU01引脚 | 颜色建议 | 说明 |
|------------|----------|----------|------|
| 3.3V | VCC | 红色 | 供电（必须3.3V） |
| GND | GND | 黑色 | 接地 |
| TX | RX | 绿色 | 数据发送（交叉） |
| RX | TX | 白色 | 数据接收（交叉） |

### 3.3 接线步骤

1. **准备工作**：确认所有设备断电状态
2. **连接电源**：CP2102-3.3V → BU01-VCC
3. **连接地**：CP2102-GND → BU01-GND
4. **连接串口**：CP2102-TX → BU01-RX（交叉！）
5. **连接串口**：CP2102-RX → BU01-TX（交叉！）
6. **连接USB**：CP2102-USB → USB集线器 → 电脑

### 3.4 注意事项

- ⚠️ **严禁接5V**：BU01必须使用3.3V供电
- ⚠️ **串口交叉**：TX必须接RX，不能直连
- ⚠️ **防静电**：操作时建议佩戴防静电手环

## 7. 业务系统对接

### 7.1 对接架构

```
┌─────────────────────────────────────────────────────────────────┐
│                      业务系统对接架构                           │
├─────────────────────────────────────────────────────────────────┤
│                                                               │
│  业务收银系统          UWB定位系统          工作人员/设备       │
│       │                     │                     │            │
│       │  开台/关台事件      │                     │            │
│       │────────────────────→│                     │            │
│       │                     │  定位查询           │            │
│       │                     │←────────────────────│            │
│       │  服务提醒          │                     │            │
│       │←────────────────────│                     │            │
│       │                     │                     │            │
└─────────────────────────────────────────────────────────────────┘
```

### 7.2 对接方式

#### 7.2.1 API轮询方案（推荐）

由于业务系统为外采系统，数据库仅开放只读权限，采用**API轮询+中间数据库**方案：

```
外采业务系统 ←(只读查询)─ 自建中间数据库 ─(事件触发)→ UWB定位系统
                           │
                           ↓
                    定时任务同步
```

**方案优势**：
- ✅ 不影响原有业务系统
- ✅ 数据隔离，安全性高
- ✅ 实现简单，无需业务系统配合
- ✅ 中等实时性，满足业务需求

**轮询配置建议**：
| 参数 | 值 | 说明 |
|------|-----|------|
| `sync_interval` | 30秒 | 数据同步间隔 |
| `service_remind_delay` | 1800秒 | 开台后服务提醒延迟（30分钟） |
| `cleaning_delay` | 60秒 | 关台后清洁提醒延迟（1分钟） |

#### 7.2.2 数据同步流程

```
1. 定时任务调用业务系统API获取订单状态
2. 检测新打开的订单，创建 room_open 事件
3. 检测已关闭的订单，创建 room_close 事件
4. 事件按配置的延迟时间触发
5. UWB系统处理事件，分配任务给就近人员
6. 任务完成后通知业务系统
```

#### 7.2.3 数据库对接（备选）

如业务系统支持写入权限，可考虑共用数据库方案，但当前场景不适用。

### 7.3 事件触发机制

#### 7.3.1 事件类型

| 事件类型 | 触发条件 | 说明 |
|----------|----------|------|
| `room_open` | 包房开台 | 客人进入包房 |
| `room_close` | 包房关台 | 客人离开包房 |
| `service_remind` | 开台后30分钟 | 提醒服务员服务 |
| `cleaning_required` | 关台 | 提醒保洁打扫 |
| `tool_checkout` | 工具取出 | 工作人员领取工具 |
| `tool_checkin` | 工具归还 | 工作人员归还工具 |

#### 7.3.2 事件流程

```
开台事件 → 记录开台时间 → 等待30分钟 → 触发服务提醒 → 检查服务员位置 → 通知就近服务员
     │
     ↓
关台事件 → 记录关台时间 → 触发清洁提醒 → 检查保洁位置 → 通知保洁打扫
```

### 7.4 数据库扩展表

#### 7.4.1 事件表 (`events`)

```sql
CREATE TABLE events (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    event_type VARCHAR(50) NOT NULL,
    room_no VARCHAR(20),
    tag_id INT,
    tag_type ENUM('person', 'item'),
    trigger_time DATETIME NOT NULL,
    status ENUM('pending', 'processing', 'completed', 'failed') DEFAULT 'pending',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_event_type (event_type),
    INDEX idx_trigger_time (trigger_time),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 7.4.2 服务员分配表 (`service_tasks`)

```sql
CREATE TABLE service_tasks (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    event_id BIGINT NOT NULL,
    room_no VARCHAR(20) NOT NULL,
    assigned_staff_id INT,
    assigned_time DATETIME,
    completed_time DATETIME,
    status ENUM('assigned', 'in_progress', 'completed', 'cancelled') DEFAULT 'assigned',
    FOREIGN KEY (event_id) REFERENCES events(id),
    INDEX idx_room_no (room_no),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 7.6 业务场景示例

#### 7.6.1 开台服务提醒流程

**服务提醒在中间数据层实现**，通过对讲耳机推送通知：

```
1. 定时任务同步业务系统订单数据
2. 检测到新订单，创建 room_open 事件并记录触发时间
3. 等待30分钟后，系统自动创建 service_remind 事件
4. 查询附近服务员位置: GET /api/staff/nearby?room_no=K05
5. 选择最近的服务员，创建服务任务
6. 通过对讲耳机系统推送语音通知
7. 服务员确认任务，开始服务
```

#### 对讲耳机集成方案

**推荐架构**：
```
UWB定位系统 ─(HTTP/WebSocket)─→ 对讲耳机网关 ─(语音)─→ 服务员耳机
```

**通知流程**：
```
1. UWB系统生成语音通知内容
   - "服务员张三，请前往K05包房服务"
2. 调用对讲网关API发送通知
3. 网关将文字转换为语音
4. 通过无线通信发送到目标服务员耳机
5. 服务员通过耳机接收语音指令
```

**对讲系统API示例**：
```
POST /api/intercom/send
Content-Type: application/json

{
    "staff_id": 1001,
    "message": "请前往K05包房服务",
    "priority": "high",
    "room_no": "K05"
}
```

**响应示例**：
```json
{
    "success": true,
    "status": "sent",
    "staff_id": 1001,
    "timestamp": "2026-05-06 11:00:00"
}
```

#### 7.6.2 关台清洁流程

```
1. 业务系统调用: POST /api/events/trigger
   { "event_type": "room_close", "room_no": "K05" }

2. UWB系统创建 cleaning_required 事件

3. 查询附近保洁人员

4. 分配清洁任务

5. 保洁人员携带工具（带标签）进入包房

6. 系统记录工具位置，确认进入包房

7. 清洁完成后，工具归还，任务完成
```

#### 7.6.3 工具管理流程

```
1. 工作人员领取工具时: POST /api/events/trigger
   { "event_type": "tool_checkout", "tag_id": 101, "staff_id": 1001 }

2. 系统记录工具状态为"借出"

3. 工作人员携带工具移动，系统实时追踪位置

4. 工具归还时: POST /api/events/trigger
   { "event_type": "tool_checkin", "tag_id": 101, "staff_id": 1001 }

5. 系统记录工具状态为"归还"
```

## 9. 常见问题

### 7.1 串口连接问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 串口无法打开 | 驱动未安装 | 安装CP2102驱动 |
| 权限不足（Linux） | 用户无串口权限 | `sudo chmod 666 /dev/tty.SLAB_USBtoUART*` |
| 端口被占用 | 其他程序占用 | 关闭占用串口的程序 |

### 7.2 AT指令问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 无响应 | 接线错误 | 检查TX-RX交叉连接 |
| 乱码 | 波特率错误 | 设置波特率为115200 |
| 指令不识别 | 固件版本问题 | 联系供应商升级固件 |

### 7.3 定位问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 定位失败 | 基站数量不足 | 确保至少3个基站在线 |
| 精度差 | 遮挡严重 | 调整基站安装位置 |
| 坐标异常 | 坐标配置错误 | 检查基站坐标配置 |

### 7.4 数据库问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 连接失败 | 配置错误 | 检查数据库连接参数 |
| 权限不足 | 用户权限不够 | 重新授权数据库用户 |
| 表不存在 | 未初始化 | 运行服务自动创建表 |

# 木桃工具包

这是“稳定、安全、合规优先”的本地生活线索监控与跟进工具包。

当前 `0.1.0` 已作为第一期基线版本，最新实测安装包已升级到 `0.1.3`：本地服务会保存任务、线索、跟进动作和审计日志。浏览器开发模式的数据存放在 `data/mvp-db.json`，桌面 exe 模式的数据存放在客户电脑的应用数据目录。

当前二期优化开发版本为 `0.1.3`，本轮优化聚焦抖音/小红书关键字精准回复、二次批量回复、账号切换节奏、结果复盘和团队实测交付。

## 运行

```bash
npm run start
```

然后打开：

```text
http://127.0.0.1:8795
```

## 桌面 exe 模式

开发预览：

```bash
npm run desktop
```

打 Windows 安装包：

```bash
npm run build:win
```

安装包会输出到 `release/` 目录，文件名类似：

```text
木桃工具包-0.1.3-安装包.exe
```

当前最新 Windows 实测安装包：

```text
release/木桃工具包-0.1.3-安装包.exe
SHA256: 以本轮最终交付消息和本机生成结果为准
```

桌面版会在客户本机启动内部服务并打开独立窗口，不需要公网地址。客户数据默认保存在 Windows 当前用户的应用数据目录，适合私有化交付和离线演示。

## 本地授权

当前版本支持离线授权码。客户输入授权码后，本机校验签名、客户名称、套餐、到期时间和可选设备绑定。到期后仍可登录查看历史数据，但不能新增、导入、运行发现、导出或下载备份。

生成月度授权码：

```bash
npm run license:create -- --customer 客户名称 --plan 月度版 --expires 2026-06-30
```

生成绑定设备码的授权码：

```bash
npm run license:create -- --customer 客户名称 --plan 月度版 --expires 2026-06-30 --device 设备码
```

私钥保存在 `keys/license-private.pem`，不要发给客户；客户安装包只包含 `license-public.pem`。

## 公网授权中心

商业化小规模落地可先部署公网授权中心，用于平台管理员生成短授权码、续期、停用和查看设备校验记录。授权中心不接收客户线索、评论、账号登录态或业务数据。

生产运行建议：

```bash
HKT_LICENSE_CENTER=1 HKT_AUTO_LICENSE=0 HOST=127.0.0.1 PORT=8795 npm run start
```

公网入口使用 Nginx + HTTPS 反代，部署模板见 `deploy/`，完整步骤见 `docs/public-license-center-deployment.md`。

## DeepSeek 意图识别

系统内置本地意图规则；配置 DeepSeek 后，会对本地规则未命中的评论做 AI 复核，用于补充识别口语化的购买、到店、配送、团购和批发意图。

- 页面配置：进入「发现」页，在「AI 意图识别」中填写 DeepSeek API Key。
- 环境变量配置：也可以设置 `DEEPSEEK_API_KEY`、`DEEPSEEK_MODEL`、`DEEPSEEK_BASE_URL`。
- 默认接口：`https://api.deepseek.com`
- 默认模型：`deepseek-chat`
- Key 只保存在本机配置或环境变量中，不会写入安装包，也不会在页面回显明文。
- 可配置单次送检数量、每日送检上限和本地识别缓存，降低重复评论带来的 API 成本。

## 本轮升级能力

- 采集任务可在矩阵任务池直接运行，记录检查数、命中数、失败原因、最近运行时间和运行日志。
- 候选转线索和人工录入会生成去重指纹，同一来源或同内容线索会合并到原线索并追加跟进记录。
- 增加“零食 + 吉林”客户演示模式，一键生成演示评论池、采集任务、候选线索和结果报告。
- 主流程调整为客户信息采集：围绕行业、作品、作者、直播和同城评论，采集评论背后的人、IP/属地、来源作品和咨询内容。
- 增加页面可见采集入口，支持把用户授权登录后页面上可见的评论文本采集进评论池。
- 手机采集能力从当前项目主线移出，后续可作为独立分支或独立产品线承接。
- 关注账号监控补充自动巡检、追踪最新作品数、新评时效窗口和强提醒邮箱；具体作品链接会生成关联采集任务，并可在授权窗口半自动/官方 API 同步层级下到点运行。
- 线索跟进台升级为客服接待式界面：会话列表、跟进对话、AI 建议、联系方式识别和线索看板。
- AI 批量沟通先生成建议并写入内部跟进记录，高意向可批量转人工；默认不直接代替员工外发私信，后续官方号/企业号能力可走小范围受控触达。
- 增加私人号/粉丝号资产池：业务人员、销售可绑定本人私人号，记录使用人、分组、每日回复/采集上限和审计轨迹。
- 迁入 5173 项目的账号绑定逻辑：支持账号台账、平台授权页面和本机加密授权信息；授权信息只在本机加密保存，页面不回显明文。
- 新增可视化账号授权：为每个账号打开独立平台授权页面，扫码或登录后可检测页面登录态；Windows 客户不需要安装 Chrome/Edge。
- 线索跟进时可选择本次人工回复账号，系统记录账号用量并做频控拦截，不保存密码或验证码。
- 抖音关键字精准回复增加硬校验：命中评论后必须在原评论下方点击回复，停留确认输入框上下文为 `回复@目标用户: 目标评论内容` 后才发送，避免因评论滚动、展开回复或图片预览导致错挂。

## 高并发设计口径

系统后续必须按高并发设计，但平台动作不能高频无约束触达。设计原则是：系统内部高吞吐处理线索、任务、去重、AI 识别、账号池和审计；真实平台回复按账号排队、限流、人工确认或官方号小范围受控发送。生产数据库优先按 MySQL 设计，单机轻量版可保留 SQLite 过渡方案。

详细设计见 `docs/high-concurrency-design.md`。

## 云端多租户迁移口径

后续商业版按“本地 exe + 云端 API + 云端 MySQL + 多租户授权计费”设计。客户只安装 exe，不需要自己安装 MySQL；客户数据进入我们托管的云端 MySQL，由云端 API 做租户隔离、员工权限、套餐额度、审计和高并发队列。

当前代码已先落多租户底座：用户、线索、任务、回复账号、评论样本、候选线索和审计日志都开始带 `tenantId`，状态接口、导出和核心操作会按当前租户过滤。详细迁移说明见 `docs/cloud-multi-tenant-migration.md`。

## 一期 MVP 已具备

- 真实登录：管理员、运营、销售账号密码登录，服务端会话有效期 8 小时。
- 多租户底座：默认租户、用户租户身份、核心数据租户归属、状态接口和导出租户过滤。
- 跟踪对象池：店铺账号、达人账号、作品 URL、直播间、自有粉丝、互动用户、官方留资。
- 获客矩阵：支持行业意向、作品 URL、作者账号、关注账号每日新增等任务入口。
- 行业意向发现：输入行业、地区和行业目标词后，生成小红书/抖音/快手搜索入口和候选账号/作品池。
- 关注账号监控：维护小红书/抖音关注账号池，支持自动巡检、追踪最新作品和新评时效窗口；账号主页先生成待选择作品任务，具体作品链接可自动采集可见评论。
- 粉丝资产分层：区分“仅观察”“需人工确认”“可外联”，避免把陌生粉丝直接当客户线索。
- 规则来源接入：支持平台采集页、页面可见评论、直播评论、自有粉丝互动等样本进入评论池。
- 手机采集：不放在当前主线版本中，后续可单独拆分为 Open-AutoGLM 类手机助手项目。
- 留资导入：保留为备用承接能力，不作为当前主动获客主流程；当前优先验证评论客户信息采集。
- 工具需求评估：把外部“AI 获客工具安装包+教程”和数字员工演示拆成可吸收需求、官方号小范围触达和风险拦截项。
- 发现规则：先把可见评论全量沉淀到评论池，再按对象类型、平台、地区、采后识别词、屏蔽词和分数生成候选线索。
- AI 意图识别：可接入 DeepSeek，对本地规则漏掉的口语化评论做二次识别。
- 评论长度条件：发现规则可设置最少字数，例如只识别 10 字以上评论。
- 候选线索：规则发现后进入待确认队列，可人工转入线索池。
- 任务创建：平台、来源、地区、采集模式、采后识别词、屏蔽词、线索上限；作品/账号/合集类任务默认先全量采集。
- 任务审核：待审核、合规评估、已通过。
- 任务运行日志：记录任务创建、审核和运行状态变化。
- 线索补录：作为采集失败或人工测试的兜底，不作为主动获客主流程。
- 线索筛选：按关键词、状态、意向、平台筛选。
- 线索详情：查看来源、状态、建议动作、跟进记录。
- 跟进备注：在线索详情里追加人工跟进记录。
- 线索跟进：分配、屏蔽、确认跟进。
- 审计日志：创建任务、处理线索、导出线索都会留痕。
- CSV 导出：线索库可导出为 `leads.csv`。
- 角色切换：管理员、运营、销售三个内部试用角色。
- 数据备份：管理员可下载完整 JSON 备份。

发现规则默认不会执行自动触达。系统先生成候选线索，由人工转入线索池；后续若接入官方号、企业号或平台允许的消息能力，可在限额、时间窗、审核、暂停和审计约束下做小范围受控触达。

## 产品边界

本项目不设计绕过平台风控、无上限批量私信、模拟人工规避检测、抓取非授权个人信息等功能。

推荐方向是：

- 官方接口、授权数据或人工导入接入
- 官方号、企业号或平台允许能力下的小范围受控触达
- 本人或员工授权账号的平台授权页面与本机加密登录态管理
- 手机应用可见评论辅助整理后续单独拆分，不作为当前项目主流程
- 公开互动线索监控
- 店铺、达人、作品、直播间观察，小红书笔记/评论可作为第一批测试渠道
- 自有粉丝和互动用户沉淀
- 行业目标词用于找入口，采后识别词与屏蔽词用于二次筛选
- 人工确认后的跟进
- 评论监听后的内部通知
- 工具教程、安装运维和渠道模板管理
- 线索留存、分配、转化统计
- 权限、审计、数据脱敏和风控策略

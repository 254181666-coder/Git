# 木桃工具包项目排期

## 定位

面向本地生活、零售、服务行业的本地线索监控与跟进工具包。系统帮助团队发现公开互动中的潜在需求，沉淀线索，提醒员工跟进，并记录转化结果。

## 安全边界

必须做：

- 使用官方接口、授权账号、人工导入或用户主动留资作为数据来源。
- 每条线索保留来源、时间、处理人、处理动作。
- 外联动作默认进入待审批或人工确认。
- 存储最小必要信息，敏感字段脱敏展示。
- 提供删除、导出、审计记录和权限控制。

不做：

- 不做绕过平台风控的自动私信、自动关注、自动评论。
- 不做批量抓取未授权个人信息。
- 不做多账号规避检测、设备指纹伪装、验证码绕过。
- 不做“保证不封号”的对抗式自动化。

## 一期 MVP，2-3 周

目标：做出可演示、可内部试用的业务闭环。

- 项目控制台：核心指标、模块状态、风险提示。已完成。
- 任务配置：平台、关键词、地区、触发词、屏蔽词、来源类型。已完成本地版。
- 线索库：来源、意向、状态、负责人、备注、跟进记录。已完成本地版。
- 人工跟进台：建议话术、确认发送、完成记录。已完成基础闭环。
- 基础导出：线索 CSV/Excel 数据结构。已完成 CSV。
- 审计日志：谁在什么时候查看、导出、处理了哪条线索。已完成基础留痕。

当前一期 MVP 使用本地 JSON 文件持久化，适合内部演示和流程确认。进入多人使用前，需要替换为数据库、加登录权限和备份策略。

- 增加内部角色：管理员、运营、销售。
- 增加基础权限：管理员可下载备份，管理员和运营可导出线索，销售可处理线索。
- 增加线索筛选：关键词、状态、意向、平台。
- 增加任务运行日志：任务创建、审核、状态变化有记录。
- 增加完整数据备份：导出 `mvp-backup.json`。

下一步建议优先做真实登录、数据库、备份恢复、线索详情页和跟进备注编辑。

## 一期第三步，已完成

- 增加账号密码登录，替换页面上的直接角色切换。
- 增加服务端会话，Cookie 仅保存会话 ID。
- 用户密码使用 PBKDF2 哈希存储，不在数据文件中保存明文密码。
- 增加线索详情页：来源、状态、内容、建议动作、跟进记录。

下一步建议做数据库替换和备份恢复。当前 JSON 持久化适合单机试用，团队多人并发使用前应切换云端 MySQL；单机轻量版可保留 SQLite 过渡方案。

- 增加粉丝资产分层：观察对象、互动用户、自有粉丝、官方留资、人工导入。

## 一期第五步，已完成

- 增加发现规则：对象类型、平台、地区、触发词、屏蔽词、最低分。
- 增加候选线索队列：发现后只进入候选，不自动触达。

- 明确触达边界：候选可触达，但必须由人确认，不做批量 Agent 自动触达。

## 三期商业版，6-10 周

目标：支持多门店、多员工、多渠道统计。

- 多门店/多项目空间。
- 转化漏斗：发现、有效、已联系、已加私域、成交。
- 话术库与审批流。
- CRM/企微客户库对接。
- 平台合规能力扩展。
- 异常检测：频率过高、敏感词、重复触达、投诉风险。

## 核心验收标准

- 稳定性：任务、线索、审计数据不丢失，可恢复。
- 安全性：敏感信息最小化、脱敏、权限隔离、导出可追踪。
- 合规性：不依赖平台规则对抗，不把自动外联作为默认能力。
- 可运营：运营人员能配置任务，销售能跟进线索，管理层能看结果。

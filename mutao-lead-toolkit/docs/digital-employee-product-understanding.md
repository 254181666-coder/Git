# 数字员工获客产品理解

来源：`/Users/ann/Desktop/数字员工.docx` 逐字稿 + 会议截图复盘。

## 一句话定位

项目不是普通客服系统，而是“多平台评论线索雷达 + 账号触达工作台”。核心目标是从抖音、小红书、快手、视频号等公开互动场景里找到刚表达需求的人，再通过官方号、授权账号或人工确认方式小范围触达并导入私域。

## 核心业务背景

- 客户行业以本地生活、零售、零食批发、同城配送、加盟、团购等为代表。
- 目标客户常常已经出现在同行作品、直播间、评论区、作者主页、行业搜索结果里。
- 工具价值不是“采一堆评论”，而是快速识别评论背后的人：谁刚咨询、在哪个地区、来自哪个作品、是否可私信、是否已关注、是否可导入私域。
- 获客速度很重要。截图和逐字稿都强调 1 分钟内、1 秒内、实时/近实时评论匹配，因为刚评论的人回复率最高。

## 截图体现的功能模块

1. 首页智能入口
   - 支持选择平台，例如抖音、小红书、快手、视频号、微信等。
   - 输入行业词后做智能分析，生成推荐词法和任务入口。

2. 行业作品意向匹配
   - 输入行业关键词，例如零食零售、零食批发、健康零食、零食加盟、零食电商。
   - 配置触发词词库，例如多少钱、联系、获取、加盟、费用、咨询、推荐等。
   - 选择执行账号和目标数量，例如 200 人。
   - 展开高级设置后可配置综合排序、点赞最多、最新发布、搜索范围、用户活跃时间、是否平分关键词、是否去重。

3. 任务看板
   - 任务卡展示任务名称、ID、打开网址、标签、目标人数、完成进度、状态。
   - 右侧看板展示当前任务数和匹配详情。
   - 匹配详情展示用户头像、昵称、评论内容。

4. Agent 互动助手
   - 互动类型包括点赞、关注、私信、评论。
   - 支持多条私信/评论话术。
   - 截图明确提示“二次私信仅支持企业号使用，发送第一条消息之后紧接着发送第二条”。
   - 这说明产品不是完全不能自动触达，而是需要区分账号类型、平台能力和触达边界。

5. 洞察/新评速递
   - 新评速递通过作品链接实时匹配作品新评论。
   - 作者动态速递通过作者链接监控新作品。
   - 可配置 IP 地址、屏蔽词、是否执行 Agent 互动、打开浏览器等。

6. 账号授权
   - 支持抖音、小红书、快手账号授权。
   - 账号卡展示账号状态、使用百分比、本地 IP、每小时请求量、每日动作额度。
   - 支持 IP 配置和新增账号授权。

## 必须保留的小范围触达能力

前一版理解里“系统不做自动私信/评论”过于绝对。更准确的边界应是：

- 不做无约束的大批量自动触达。
- 可以做官方号、企业号、平台允许能力下的小范围触达。
- 可以做本人/员工授权账号下的低频触达，但必须有额度、间隔、任务审计、失败暂停和人工可接管。
- 私信、评论、关注、点赞应拆成不同动作类型，每个平台分别配置是否允许。
- 非官方号或平台限制较严时，系统应退回“打开定位 + 复制话术 + 人工确认发送”。

建议把触达分成三档：

1. 人工确认触达
   - 系统打开来源作品/用户主页，复制定位词和话术。
   - 员工手动关注、私信、评论或留资。

2. 半自动受控触达
   - 系统排队、限频、填充话术、打开发送界面。
   - 员工最后确认发送。

3. 官方号小范围自动触达
   - 仅对官方接口、企业号、平台留资工具、平台明确允许的消息能力开放。
   - 必须配置每日上限、单任务上限、随机间隔、工作时间窗、二次消息规则、黑名单词、暂停条件和审计日志。

## 我们当前项目已覆盖的部分

- 评论池、批量评论导入、可见评论导入。
- 发现规则：平台、地区、触发词、屏蔽词、最少字数、最低分。
- 候选线索和转入线索池。
- 线索去重合并。
- 账号池、账号授权、本机加密登录态。
- 作品链接采集和来源定位。
- 评论作者、来源评论、来源作品、IP/属地字段。
- 线索跟进、复制话术、保存记录、账号频控。

## 明显缺口

- 任务类型还没有完整对应截图里的“匹配、互动、洞察、半自动、同城、直播”。
- 缺少官方号/企业号接入的触达策略层。
- 缺少“Agent 互动助手”式动作配置：点赞、关注、私信、评论、多话术、二次私信、动作顺序。
- 缺少“新评速递/作者动态速递”的近实时监控队列。
- 缺少每个账号的健康百分比、请求量、动作额度、自动暂停和恢复策略。
- 缺少 IP 配置、账号与 IP 绑定展示、平台差异化限额展示。
- 缺少官方留资卡片/获客链接承接模块。

## 建议落地优先级

1. 产品口径修正
   - 从“完全不自动触达”改为“禁止大批量违规触达，支持官方号/授权账号的小范围受控触达”。

2. 任务模型补强
   - 新增任务大类：匹配、互动、洞察、同城、直播。
   - 新增任务字段：排序方式、搜索范围、活跃时间窗、平分关键词、提醒邮箱、是否执行互动、打开浏览器。

3. 人资产字段补强
   - 用户主页、平台用户 ID、头像、昵称、IP/属地、来源作品、来源评论、是否关注、是否可私信、是否已互动、最近互动时间。

4. 互动助手二期
   - 动作类型：点赞、关注、私信、评论。
   - 话术库：首条话术、二次私信、评论话术、留资卡片。
   - 执行策略：人工确认、半自动确认、官方号自动。

5. 官方号接入
   - 先做配置入口和策略模型，不急着承诺所有平台都可自动发。
   - 抖音企业号、小红书专业号、快手商家号、视频号/企微链路分别评估官方能力。

6. 风控和审计
   - 每账号每日动作上限、每小时请求量、随机间隔、失败暂停、异常提示、操作日志、导出审计。

## 最终产品边界

系统应做：找人、筛人、排队、提醒、定位、生成话术、受控触达、留资承接、私域沉淀、审计复盘。

系统不应做：绕过验证码、绕过平台风控、代理 IP 规避、非授权抓取个人隐私、无上限批量私信、无人工/官方授权的陌生人批量骚扰。

## 今日目标

## 今日完成

1. 线索跟进台升级为客服接待式工作台
   - 会话列表、聊天式跟进记录、AI 建议、联系方式识别、线索看板。
   - 支持 AI 批量生成建议，高意向线索批量转人工。
   - 仍保持人工确认，不自动私信、不自动评论。

2. 新增私人号资产池
   - 可登记多个用于查看评论和人工回复的账号。
   - 支持账号分组、使用人、启停、每日回复上限、每日采集上限。
   - 线索跟进时可选择本次回复账号，系统会记录用量并做频控拦截。

3. 迁入 5173 项目的账号绑定口径
   - 支持三种绑定方式：仅台账、应用内置登录窗口、本机加密登录态。
   - 登录态只在本机加密保存，页面和状态接口不回显明文，也不回传加密串。
   - 支持检测绑定、清除绑定、账号审计留痕。

4. 新增可视化登录授权
   - 为每个账号打开应用内置登录/采集窗口，不读取用户日常浏览器资料，也不要求 Windows 客户安装 Chrome/Edge。
   - 用户在内置窗口里扫码/登录平台。
   - 回到系统点击“检测并保存”后，系统读取内置窗口登录态，做页面级登录检测，并在本机加密保存。
   - 当前支持抖音、小红书、快手的基础平台识别；真实检测效果需要明天用实际账号验证。

## 本轮本地预览调整

- 根据复盘补齐核心产品理解：评论只是入口，真正要沉淀和跟进的是评论背后的“人”。主链路应按“公开评论命中意向 -> 反查评论用户/主页 -> 记录评论 IP/属地或地址线索 -> 由人工判断是否反关/进入可私信状态 -> 私信、评论或私域留资承接”设计。
- 线索结构补强为以人为中心：评论导入支持“作者｜IP/属地｜评论”三段格式；候选线索和正式线索会保留评论作者、来源评论、来源作品、评论 IP/属地，并在接待台、详情、导出和「打开来源定位」定位词里展示。
- 触达边界重新校准：不是完全不做自动触达，而是不做无约束的大批量自动触达。默认链路仍由人工确认；后续可增加官方号/企业号/平台允许能力下的小范围受控触达，必须带额度、间隔、审核、暂停和审计。
- 暂时隐藏前端「留资导入」入口，首页不再提示配置留资导入；后端接口和数据结构保留，后续需要时可恢复。
- 左侧「评论识别」改名为「评论池识别」，更贴近实际用途：评论先进入评论池，再由规则识别候选线索。
- 「线索录入」改为「线索补录」，定位为备用手工补录，不作为主动获客主链路。
- 线索跟进区明确区分「复制回复」和「保存记录」：复制后由人工到平台评论区发送，保存只写入本地跟进历史，不会自动同步到对方评论区。
- 新增半自动回复定位雏形：评论样本、候选线索、线索会尽量保留来源作品链接；线索跟进页增加「打开来源作品」「复制定位词」「打开来源定位」。
- 「打开来源定位」会用选中的回复账号打开来源作品，并复制评论作者/评论内容定位词；正式桌面客户端可调用内置登录窗口，本地网页预览模式只能查看按钮和数据，不能真正打开内置平台窗口。
- 产品边界调整为“系统负责筛选、排队、定位、填充和记录；平台发送仍由人工确认”，不做后台批量自动评论。
- 根据腾讯会议逐字稿补齐截流任务核心参数：执行账号、咨询时间窗、动作方式、随机间隔、互动话术、提醒邮箱、自动去重。任务列表会展示这些参数，后续可围绕它做“评论出现后短时间内提醒/排队/定位”的闭环。
- 逐字稿理解的竞品目标：行业/作品/作者维度找公开评论用户，按地区/IP、触发词、屏蔽词和去重筛选；多账号分摊监控作品；命中后强提醒并以私信/评论/留资卡片承接。当前项目先实现合规版半自动截流，不做后台批量自动发送。
- 下一步如果要更贴近竞品链路，应优先做“评论用户主页/用户ID提取、反关状态、是否可私信、私信入口定位、反关后的跟进状态”这组字段和按钮，而不是继续只堆评论采集入口。
- 根据截图复盘补齐竞品模块：匹配、互动、洞察、半自动、同城、直播；行业作品意向匹配支持排序方式、搜索范围、用户活跃时间、是否平分关键词；Agent 互动助手支持点赞、关注、私信、评论、多条话术和企业号二次私信；账号授权页展示本地 IP、请求量、动作额度和账号健康百分比。

   - 如果是在抖音搜索结果弹窗中打开作品，直接点击“复制作品链接”，系统会把 `modal_id` 链接转为正式作品链接。
   - 先直接运行对应作品 URL 采集任务，确认系统能自动读取内置采集窗口当前可见评论。
   - 如果自动读取失败，再用「浏览器可见采集」入口粘贴少量真实可见评论作为兜底验证。
   - 运行发现规则，看候选线索是否生成。

2.5 验证手机采集入口
   - 进入「手机采集」页。
   - 用数据线连接安卓手机并开启 USB 调试。
   - 登记测试设备，点击“检测连接”。
   - 创建美团或抖音/小红书手机采集任务。
   - 把手机页面可见评论整理为文本导入，确认进入「评论池识别」样本池。

3. 留资导入暂不验证
   - 当前前端入口已隐藏，先聚焦主动获客、评论采集、线索识别和人工跟进闭环。

4. 再验证线索跟进
   - 将候选线索转入线索池。
   - 进入「线索」页。
   - 选择回复账号。
   - 让 AI 生成建议话术。
   - 点击「打开来源定位」，让系统打开来源作品并复制评论定位词。
   - 人工在平台页确认对应评论，粘贴话术并发送。
   - 回系统保存跟进记录。

5. 最后验证频控和审计
   - 把某个测试账号每日回复上限调低，例如 1。
   - 连续记录 2 次回复，确认第 2 次被拦截。
   - 检查账号卡片和审计日志是否有操作记录。

## 明天需要重点观察

- 内置登录/采集窗口是否能稳定打开抖音/小红书登录页。
- 扫码成功后，“检测并保存”能否识别登录态。
- 平台昵称是否能正确识别，至少抖音应优先验证。
- 登录态保存后，页面和接口是否仍不回显明文。
- 人工真实回复后，线索状态、账号用量、跟进记录是否一致。
- 如果平台出现安全验证、二维码过期、登录失败，提示是否清晰。
- 多租户字段是否影响现有页面、导出、备份和线索跟进流程。
- 后续切云 MySQL 前，是否需要保留本地 JSON 演示模式和云端生产模式两套配置。

## 当前风险边界

- 系统一期默认不做后台自动关注、自动评论、自动私信。
- 后续若接入官方号/企业号/平台官方能力，可开放小范围受控触达，但必须按账号、任务、时间窗和动作类型限额，并保留人工接管和审计日志。
- 系统不做多开规避、验证码绕过、代理 IP 规避。
- 可视化授权只服务于本人或员工授权账号的本机管理。
- 明天真实回复仍应优先由人工在平台页面确认发送；官方号触达能力需单独评估平台接口和账号资质。
- 高并发只用于内部任务处理、线索流转和多人协作；真实平台回复必须按账号排队、限流和审计。

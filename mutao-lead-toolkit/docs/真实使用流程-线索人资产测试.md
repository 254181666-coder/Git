# 真实使用流程：客户信息采集测试

更新时间：2026-05-25

## 测试目标

验证主链路是否围绕“采集客户信息”运行：从行业/作品/作者/直播/同城评论里找到刚表达需求的人，沉淀平台账号、头像昵称、评论、IP/属地、来源作品，再进入线索人资产和跟进队列。

## 核心口径

当前项目不把“导入线索”作为主流程。真实业务里客户主动留微信、手机号或表单留资的概率很小，所以主流程应是：

1. 找到同行作品、作者账号、直播间或同城关键词。
2. 采集评论区里刚表达需求的人。
3. 按采后识别词、屏蔽词、IP/属地、活跃时间筛选。
4. 形成线索人资产。
5. 人工或官方号小范围触达。

“全量采集”的口径：对当前作品页或授权窗口里能加载出来的可见评论尽量拉满，当前系统上限是单任务采集 5000 条、评论池识别 50000 条；它不是平台后台全库抓取，也不能绕过平台登录、加载和风控限制。

“行业目标词”和“采后识别词”的口径：行业目标词只用于在行业意向发现里找到作品、作者或用户入口；作品、作者、账号和合集任务不需要先填词，系统应先把可见评论保存进评论池，再用识别词、屏蔽词、地区、字数和 AI 判断生成候选线索；未命中识别词的评论也会保留，后续可换词或启用智能识别重新筛。

## 流程一：准备采集账号

1. 进入“回复账号池”。
2. 新增抖音、小红书、快手或视频号账号。
3. 业务人员本人账号先选“粉丝号/私人号”；如果有企业号或官方号，再选“官方号/企业号”。
4. 在桌面端点击“一键登录授权”，完成扫码或登录。
5. 检查账号容量：每日采集、每日触达、每小时请求量、账号健康百分比。

## 流程二：行业入口找作品和作者

1. 进入“客户信息采集”。
2. 选择“行业作品意向匹配”。
3. 设置：
   - 行业：零食
   - 地区：吉林或长春
   - 采集模式：全量采集或定额采集
   - 目标采集量：测试可用 200 至 500，正式任务可提高
   - 行业目标词：多少钱、地址、配送、团购、批发、怎么拿货
   - 屏蔽词：互粉、广告、抽奖、同行
4. 创建任务后，系统会生成平台搜索入口和候选作品/作者模板。
5. 打开搜索入口，选择具体作品或作者主页，再复制链接填入作品任务或作者任务。

## 流程三：精准作品采集

1. 进入“精准作品意向匹配”。
2. 粘贴具体作品链接。
3. 平台会按链接自动识别；也可以手动切换平台后再粘贴链接。
4. 采集模式默认“全量采集”，先把当前作品页可见评论尽量入池。
5. “采后识别词”可以先不填，后续在评论池或发现规则里再筛。
6. 运行任务后，系统会尝试读取平台采集页里当前页面可见评论。
7. 命中的评论会进入评论池，再按规则生成候选客户。

## 流程四：作者/账号监控

1. 进入“关注账号每日新增”。
2. 填入同行或目标作者主页链接；如果已经拿到具体作品链接，优先填具体作品链接。
3. 设置：
   - 追踪最新作品数：建议 10
   - 新评时效窗口：建议 1 分钟
   - 自动巡检：开启
   - 巡检间隔：24 小时，测试时可缩短
   - 自动化层级：自动记录巡检、授权窗口半自动或官方 API 同步
4. 若链接是具体作品，系统会自动生成关联采集任务。
5. 自动化层级选择“授权窗口半自动”或“官方 API 同步”时，到点会尝试运行关联采集任务，读取授权窗口里当前作品页可见评论。
6. 若链接是账号主页或搜索入口，系统会生成“需选择作品”的关联任务。真实测试时先打开最新作品，把作品链接填入作品任务或监控链接后再采集。
7. 监控卡片会显示本次评论、本次候选、累计采集和自动采集说明。

## 流程五：线索人资产查看

1. 进入“线索人资产”。
2. 查看顶部统计：
   - 线索人资产
   - IP/属地
   - 可私信
   - 待触达
3. 使用筛选：
   - 全部人资产
   - 有 IP/属地
   - 可私信
   - 待触达
   - 已沉淀
4. 每个客户信息应重点看：
   - 平台账号或昵称
   - 来源评论
   - 来源作品
   - IP/属地
   - 是否可私信
   - 是否已反关
   - 是否已触达

## 流程六：触达与标记

当前“反关待确认、私信待确认、待触达”是状态，不是自动执行平台动作。

真实操作：

1. 在线索人资产里点击“打开主页”或“去跟进”。
2. 人工判断这个人是否真实、是否符合地区和需求。
3. 在平台完成反关后，回系统点“标记已反关”。
4. 确认可以私信后，点“标记可私信”。
5. 完成私信、评论或官方号触达后，点“标记已触达”。

## 流程七：抖音关键字精准评论回复

适用场景：用户已授权登录抖音，打开具体作品评论区后，按关键词找到评论，并在原评论下方回复。

1. 作品链接优先使用具体作品或 `modal_id` 链接，不使用搜索页或作者主页直接执行回复。
2. 打开作品后先进入评论区；如果评论区没有展开，先用页面评论按钮或快捷键打开评论区。
3. 按命中关键词筛选评论内容，例如“抖音精选”。
4. 每条命中评论必须记录：评论用户、评论内容、IP/属地、来源作品链接。
5. 点击命中评论下方的“回复”后，必须稍作停留，等待输入框切换完成。
6. 发送前必须二次确认输入框上下文：应出现 `回复@目标用户: 目标评论内容`。
7. 如果 `回复@` 的用户或评论内容与当前命中评论不一致，立即停止本条发送，关闭或重选回复框后重新定位。
8. 确认无误后输入回复话术，再点击红色发送箭头或回车发送。
9. 发送后等待平台反馈；看到“已发布”、评论数增加或结果卡记录成功，才算本条完成。
10. 成功和未成功都进入结果复盘；未成功必须保留失败原因，方便人工补操作。
11. 全量回复验收：同一作品内当前可见评论应尽量深扫到上限，先按“平台用户 + 评论内容”去重，再生成回复队列；相邻两条评论都命中关键词时，两条都必须进入队列并分别执行。
12. 速度验收：允许缩短普通执行间隔，但不能取消点击“回复”后的停留确认；宁可单条失败进入复盘，也不能因为抢速度发错对象。
13. 作品直达模式验收：全量爬取完成后，后续逐条回复必须复用当前作品页，不能每回复一条就刷新一次页面；只允许重新确认评论区、目标回复按钮和回复框上下文。
14. 全量爬取价值验收：全量扫描用于找全命中评论；只有命中关键词的去重评论进入评论池和精准回复队列，未命中的评论不保留。复盘只展示命中评论，并区分已成功和未成功。
15. 未成功二次操作：结果复盘中未成功评论必须可二次处理，支持批量重试未成功、单条重新自动回复、打开来源人工处理、标记已处理和忽略；二次自动回复复用原来源作品、原评论作者、原评论内容和原回复话术。批量重试按平台分组，抖音和小红书可并发，各平台内部串行。

已实测规则：不能只按第几个“回复”按钮或视觉位置发送。抖音评论区会因为滚动、展开回复、图片预览或虚拟列表变化导致按钮序号变化；唯一可靠门槛是发送前确认 `回复@目标用户` 和评论片段。

## 流程八：小红书关键字精准评论回复

适用场景：用户已授权登录小红书，打开具体笔记页后，按关键词找到评论，并在原评论下方回复。

1. 小红书复制链接常见格式为 `https://www.xiaohongshu.com/explore/笔记ID?xsec_token=...&source=web_explore_feed`。
2. 这类 `/explore/` 链接是具体笔记链接，不是搜索页；系统应保留 `xsec_token`、`xsec_source`、`source`、`channel` 等参数。
3. 打开笔记页后，先深扫当前可见评论，按“平台用户 + 评论内容”去重后生成回复队列。
4. 每条命中评论必须记录：评论用户、评论内容、来源笔记链接。
5. 点击命中评论下方的消息气泡或“回复”后，必须稍作停留，等待底部输入框切换为回复状态。
6. 发送前优先用刚点击的“回复”按钮附近上下文确认目标用户和评论片段；底部输入框只作为“已进入回复/发送态”的确认，因为小红书底部输入框父级可能混入整页评论文本。
7. 如果点击锚点上下文和当前命中评论不一致，立即停止本条发送，重新定位消息气泡。
8. 确认无误后输入回复话术，再按回车或点击“发送”。
9. 发送后等待平台反馈；成功、失败、未找到气泡、回复锚点不一致都要进入结果复盘。

已实测规则：小红书评论页的“回复”可稳定打开底部 `contenteditable` 输入框，草稿可写入，`取消` 可清空草稿；但取消后页面仍可能保留“发送/取消”面板状态，所以不能只靠按钮存在判断发送或取消闭环。

真实发送验证：在笔记 `69f455fc000000003601b011` 中回复 `要疯狂到发疯`，内容 `我也想去`，页面回显 `小红薯86119E55 回复 要疯狂到发疯 : 我也想去`，评论数从 `524` 增加到 `525`。因此发送成功判定应优先看“新回复文本出现”或“评论数增加”，输入框清空只能作为辅助信号。

## 双平台执行规则

1. 抖音和小红书可以同时执行，但必须拆成两个平台任务。
2. 平台之间可以并发；同一平台、同一账号、同一真实浏览器会话内必须串行。
3. 每个平台使用自己的授权账号和页面控制通道，避免两个任务同时抢一个页面焦点。
4. 默认建议：抖音并发 1、小红书并发 1、全局最多 2 路；任一平台出现登录、验证码、风控或连续失败，只暂停对应平台，不影响另一平台。

## 自动化边界

现在可自动：

- 到点触发账号监控巡检。
- 自动记录巡检任务日志，并为账号监控生成关联采集任务。
- 对具体作品链接，在“授权窗口半自动”或“官方 API 同步”层级下，到点尝试运行评论采集。
- 作品任务可尝试读取授权窗口中的页面可见评论。
- 采集结果进入评论池后，可自动跑规则识别候选客户。

下一步可继续做：

- 作者主页自动发现最新作品。
- 自动进入最新作品并读取新增评论。
- 邮件强提醒，把账号、作品、评论用户和任务编号发给运营。
- 官方号/企业号按平台允许能力做小范围、限频、可审计触达。

不应承诺：

- 无授权后台抓取平台数据。
- 绕过验证码或风控。
- 无限制批量私信、关注、评论。
- 非官方能力下的稳定自动私信。

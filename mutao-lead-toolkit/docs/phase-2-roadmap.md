# 木桃工具包二期优化路线

当前 `0.1.0` 安装包作为第一期基线版本保留。二期在此基础上继续优化，目标是从“可演示、可试用”推进到“真实客户可持续使用”。

当前二期开发版本：`0.1.3`。第一批优化先回到一期的简洁主流程；评论采集页降级为内部采集池，客户主入口集中到 Agent 执行，结果复盘放在 Agent 执行下面。Agent 执行页保留“自动评论任务”和“关键字精准回复”，统一进入执行/完成卡片。本机诊断与排障信息保留为内部能力。

## 版本节奏

- `0.1.x`：第一期基线后的修复版，优先处理安装、启动、显示、采集稳定性和明显流程问题。
- `0.2.x`：二期功能增强版，逐步加入更完整的任务运行、通知、去重、账号频控和导入模板。
- `1.0.0`：稳定交付版，要求真实 Windows 场景验证通过，具备清晰安装、授权、备份、恢复和运维流程。

## 第一优先级：真实可用

- Windows 安装包真实安装、启动、卸载、升级测试。
- 本机诊断页：展示版本、数据目录、业务数据文件、授权文件、崩溃日志、安装包 SHA256 和最近运行记录，支持下载排障信息。
- 采集线索模块收敛：客户可见入口只保留账号池、Agent 执行和结果复盘；评论采集、采集结果识别、留资导入、监控对象、运行日志、诊断和审计先作为内部能力隐藏。
- 手机采集模块从当前项目主线移出，后续作为单独分支或单独产品线承接 Open-AutoGLM / 手机助手能力。
- 平台授权页面稳定性测试，覆盖抖音、小红书等首批渠道。
- 作品链接评论采集稳定性，重点验证评论弹层、滚动加载、重复评论过滤、关键词命中和失败提示。
- 精准评论回复流程验证，确保边采集边回复、关键词筛选命中评论、打开来源作品、定位评论人/评论内容、在该评论下方挂载回复、人工确认、账号额度消耗、已回复成功/未回复成功记录和完成卡片闭环顺畅。
- 崩溃日志、启动失败提示和客户排障说明补齐。

## 第二优先级：运营效率

- 线索去重二期：同一用户、同一作品、同一联系方式、相似评论的合并和人工拆分。
- 多渠道批量导入模板：评论作者、IP/属地、评论内容、来源作品、平台、时间字段标准化。
- 任务运行页二期：运行队列、取消运行、失败重试、历史日志和单任务结果明细。
- Agent 执行台二期：围绕自动评论和关键字精准回复呈现，支持作品链接一键执行、复制话术、标记完成、账号频控提醒、待审核队列、失败复盘和完成卡片筛选。
- 企业微信或内部通知：候选线索命中后推送运营或销售人员。

## 第三优先级：商业化底座

- MySQL 数据库替换或云端数据层，提升多人并发、备份恢复和查询稳定性。
- 多租户授权后台：客户、套餐、到期时间、额度、员工账号和审计统一管理。
- 账号绑定二期：账号健康度、授权过期提醒、平台差异化检测和账号级任务排队。
- 小范围受控触达：仅在官方号、企业号或平台允许能力下，带额度、间隔、审核、暂停和审计。
- Windows 代码签名，降低客户安装时的系统安全提示。

## 持续边界

- 不做验证码绕过、风控对抗、逆向私有接口。
- 不做无上限批量私信、关注、评论或自动触达。
- 不采集非必要个人信息，敏感字段优先脱敏和最小化存储。
- 所有外联动作优先人工确认；如接入官方能力，必须限频、可暂停、可追踪。
